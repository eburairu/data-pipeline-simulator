# Data Pipeline Simulator Constitution

## Core Principles

### 1. 言語とコミュニケーション (Language & Communication)
*   **日本語第一**: 全ての出力（質問、回答、プラン、思考、PRの文章）、コードコメント、ドキュメントは日本語で行う。
*   英語で指示があった場合でも、回答は日本語で行う。

### 2. コード品質とスタイル (Code Quality & Style)
*   **コメント**: コード内のコメントは全て日本語で記述する。既存の英語コメントも発見次第、日本語に翻訳・更新する。
*   **型安全性**: TypeScript の機能を最大限活用し、`any` の使用を避ける。
*   **コンポーネント設計**: React コンポーネントは再利用性と可読性を重視して設計する。

### 3. バージョン管理 (Version Control)
*   **Conventional Commits**: コミットメッセージは Conventional Commits に従う。
    *   **ヘッダー (Subject)**: 英語で簡潔に記述する（例: `feat: add new feature`）。
    *   **本文 (Body)**: 日本語で詳細を記述する。

### 4. テストと検証 (Testing & Verification)
*   **テストファースト**: 可能であればテスト駆動開発 (TDD) を実践する。
*   **検証**: コード変更後は必ず動作確認を行い、その結果を記録する。

### 5. 機能統合チェックリスト (Integration Checklist)
新しいジョブタイプや主要なエンティティを追加する際は、以下の全画面・全機能との接点を必ず確認し、仕様書に含めること。
*   **Settings (設定)**: 定義の作成・編集・削除ができるか。
*   **Control (制御)**: シミュレーションの開始/停止、手動実行が可能か。
*   **Monitor (監視)**: 実行状況がリアルタイムに表示（ビジュアル・ログ）されるか。
*   **Dashboard (可視化)**: BIダッシュボードやメトリクスへの反映が必要か。

## Governance

本憲章はプロジェクトの全ての開発活動に適用される。`AGENTS.md` の指示と本憲章の内容が競合する場合は、より具体的な指示（`AGENTS.md` またはユーザーの明示的な指示）を優先するが、基本的には本憲章の原則を遵守すること。
